Picking a default: For new code, start with gpt-4.1-mini (great balance of cost, quality, 1M context) or gpt-5-mini (when you want reasoning). Step up to gpt-5 only for the hardest tasks. gpt-4o-mini remains a solid pick for high-volume non-reasoning workloads.

Reasoning vs non-reasoning: GPT-5 and o-series spend hidden "reasoning tokens" before answering. They do not accept temperature or top_p. Use reasoning_effort and verbosity instead. Use max_completion_tokens (not max_tokens) — and budget for reasoning tokens within it.

Authentication & Setup

The OpenAI API uses API keys for authentication. Follow these steps to get started with Python using the openai SDK.

1
Create an OpenAI account and API key
Visit platform.openai.com → API keys → Create new secret key. Store it immediately — it won't be shown again.
2
Set the API key as an environment variable
Never hardcode API keys in source code. Use environment variables:

export OPENAI_API_KEY="sk-proj-..."  # Linux / macOS
$env:OPENAI_API_KEY="sk-proj-..."  # Windows PowerShell

Or store in a .env file and load with python-dotenv. Add .env to .gitignore.
3
Install the SDK and initialize the client
uv add openai  or  pip install openai

The SDK automatically reads OPENAI_API_KEY from the environment:
4
Make your first API call
The SDK is synchronous by default. For production use, consider the async client: AsyncOpenAI()

Two Text APIs — Chat Completions vs Responses

OpenAI now offers two first-class text-generation endpoints. Both are supported. They differ in shape, state management, and tool support.

Dimension Chat Completions Responses
Endpoint/chat/completions/responses
ReleasedMarch 2023March 2025

Rule of thumb: For new code — start with Responses, especially for agents, reasoning models, or multi-turn. For wire compatibility with non-OpenAI providers or existing apps — Chat Completions remains the standard.

Response Anatomy — Chat Completions

Click any field to see what it means and how to use it in your code.

"id" "chatcmpl-abc123..." Unique identifier for this completion. Use for logging and debugging. Format: chatcmpl- prefix followed by a random string. metadata
"model" "gpt-4.1-mini-2025-04-14" The specific model version used. OpenAI returns a date-pinned version rather than the alias. Use for reproducibility logging. metadata
"choices[0].message.content" "An API is a..." The model's actual text response. This is what you use in your application: response.choices[0].message.content. Will be null if the model used a tool call. important
"choices[0].finish_reason" "stop" Why generation stopped. stop = normal completion. length = hit max_completion_tokens (truncated — increase the cap). tool_calls = model called a tool. Always check in production. important
"usage.prompt_tokens" 42 Number of tokens in your input (system + user messages combined). Billed at the "input tokens" rate. Prompt-caching can discount this ~50–90% for repeated prefixes. cost
"usage.completion_tokens" 87 Number of tokens in the model's response. Billed at the "output tokens" rate (usually 4–10× more expensive per token than input). max_completion_tokens caps this value. cost
"usage.completion_tokens_details.reasoning_tokens" 128 Only on GPT-5 / o-series. Hidden reasoning tokens — billed at the output rate but NOT returned to you. Add these to your max_completion_tokens budget. cost
"usage.total_tokens" 129 prompt_tokens + completion_tokens (including reasoning_tokens). Track across requests to estimate monthly costs. cost
"created" 1720000000 Unix timestamp (seconds since epoch) when the request was processed. Convert with datetime.fromtimestamp(response.created). metadata
Accessing the response in Python:
text = response.choices[0].message.content
tokens_used = response.usage.total_tokens
finish = response.choices[0].finish_reason # check for "length"!

Response Anatomy — Responses API

Different shape: output_text for plain text, or iterate over typed items in output[] for tool calls, reasoning, etc.

"id" "resp_abc123..." Pass this as previous_response_id on the next call to maintain server-side conversation state. important
"output_text" "An API is a..." Convenience: the concatenated text of all message items. The primary accessor for simple text responses. Equivalent to walking output[].content[]. important
"output" [ {type: "message", ...}, {type: "function_call", ...} ] Ordered list of typed items the model produced this turn: message, function_call, reasoning, web_search_call, image_generation_call, etc. important
"status" "completed" Lifecycle state: completed, incomplete (cap hit), failed, or in_progress (background mode). important
"incomplete_details.reason" "max_output_tokens" When status="incomplete", this tells you why — e.g. max_output_tokens or content_filter. Equivalent to finish_reason on Chat Completions. important
"usage.input_tokens" 42 Equivalent to prompt_tokens on Chat Completions. Note the renamed field. cost
"usage.output_tokens" 87 Equivalent to completion_tokens. Includes reasoning tokens when present. cost
"usage.output_tokens_details.reasoning_tokens" 128 Hidden reasoning tokens for GPT-5 / o-series. Billed at the output rate but not returned to you. cost
"previous_response_id" "resp_prev_xyz789" If this response was a follow-up, this field echoes the ID of the prior response in the conversation chain. metadata
Accessing the Responses response in Python:
text = response.output_text
tokens_used = response.usage.total_tokens
if response.status == "incomplete": print(response.incomplete_details.reason)
next_id = response.id # pass as previous_response_id on the next call
