The Three Message Roles

Every message in the Chat Completions API must have a role. The role tells the model who is "speaking" and how to interpret that message.

🧭
system
Sets the model's persona, behavioral rules, and constraints for the entire conversation. Processed first, before any user messages. Persists across all turns.
{
  "role": "system",
  "content": "You are a concise technical assistant. Answer in under 3 sentences."
}
👤
user
Input from the human — questions, instructions, or content to process. This is the primary "turn" in the conversation. Alternates with assistant messages.
{
  "role": "user",
  "content": "Explain what an API is."
}
🤖
assistant
The model's previous responses. Include these when building multi-turn conversations so the model knows what it said before. Without them, each call is independent.
{
  "role": "assistant",
  "content": "An API is an interface that allows software components to communicate."
}
Key insight: The model has no memory between API calls. To maintain conversation history, you must include all previous messages (user + assistant turns) in each request. This means context grows with every turn — and so does the cost.

Context Window Management

As conversations grow, token costs and latency increase. Use these strategies to manage context window usage in production.

Sliding Window
Keep only the last N turns. Drop the oldest user+assistant pairs when the total token count exceeds a threshold. Simple and effective.
Summarization
Periodically summarize older conversation history into a single condensed message. Trade detail for token efficiency. Good for very long sessions.
Message Trimming
Remove or truncate individual messages that exceed a per-message token budget. Use tiktoken to count tokens before building your messages array.
Stateless per Turn
For task-based apps (not chat), don't maintain history at all. Each API call is independent. Use RAG to inject relevant context instead of raw history.
Counting tokens: Use the tiktoken library (uv add tiktoken) to count tokens before making API calls.
