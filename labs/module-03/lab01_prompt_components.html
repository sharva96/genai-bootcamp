The Six Prompt Components

Click each component to expand its example. Not all components are required for every prompt.

Component 1 System Prompt Optional

Sets the model's persona, role, and behavioral constraints for the entire interaction. Think of it as the "job description" for the AI.

You are a customer experience analyst.
Classify reviews with precision.
Always respond in valid JSON only.
Do not include explanations or caveats.
Click to see example ▾
Component 2 Context / Background Optional

Provides situational knowledge the model needs but doesn't have — your domain, audience, constraints, or any relevant background information.

## Context
These reviews are from an e-commerce platform for consumer electronics.
Customers range from technical enthusiasts to casual buyers.
Reviews may contain sarcasm or informal language.
Click to see example ▾
Component 3 Examples (Few-Shot) Optional

Demonstrates the desired input→output behavior. Conditions the model's output toward your expected format and style without updating model weights.

Example 1:
Review: "Great product, fast delivery!"
Output: {"sentiment": "positive", "confidence": 0.95}

Example 2:
Review: "Broke after two days, terrible quality."
Output: {"sentiment": "negative", "confidence": 0.98}
Click to see example ▾
Component 4 Instruction Always Required

The explicit task directive — what you want the model to do. Use clear imperative verbs: Classify, Extract, Summarize, Generate, Analyze, Compare.

Analyze the following customer review and classify its sentiment.
Identify the key phrases that drive the sentiment.
Click to see example ▾
Component 5 Input Data Often Required

The content to act on — a document, code snippet, text, or data. Always delimit it clearly from the instruction to prevent prompt injection.

Review: """
The laptop runs fast and the battery lasts
all day. Setup was a breeze. However, the
keyboard feels shallow compared to my old one.
"""
Click to see example ▾
Component 6 Output Format Spec Optional

Tells the model exactly how to structure its response — JSON schema, length limits, format type, tone, and any constraints on the output.

Respond in JSON with exactly these keys:
- sentiment: "positive" | "negative" | "neutral" | "mixed"
- confidence: float 0.0–1.0
- key_phrases: array of strings
- summary: one sentence

Do not include any other text.

Weak vs Strong Prompts

See how adding components transforms a vague prompt into a precise, reliable one.

✗ Weak Prompt
Analyze this review.
✗No instruction verb — "analyze" is ambiguous
✗No output format — prose, JSON, table?
✗No context about the domain or audience
✗No data delimiters — ambiguous boundaries
✓ Strong Prompt
You are a customer analytics AI. Reviews are from an e-commerce electronics platform. Classify the sentiment of this customer review. Review: """ The laptop runs fast but the fan is loud. """ Return JSON: {"sentiment": "positive|negative|neutral|mixed", "confidence": 0.0–1.0, "key_phrases": [...]}
✓Clear persona via system prompt
✓Domain context provided
✓Input data clearly delimited with triple quotes
✓Explicit JSON output schema specified
✗ Weak Prompt
Fix the bug in my code.
✗No code provided — model has nothing to act on
✗No description of what the bug is or does
✗No language, framework, or version context
✗No expectation for explanation vs just the fix
✓ Strong Prompt
You are a senior Python engineer. The codebase uses Python 3.11 and FastAPI. Identify and fix the null pointer exception in the function below. Explain what caused it in one sentence. ```python def get_user(user_id): user = db.find(user_id) return user.name # crashes here ``` Return: fixed code block + one-sentence explanation.
✓Expert role persona set
✓Language and version context provided
✓Code delimited in a code block
✓Expected output format (code + explanation) specified
✗ Weak Prompt
Write an email declining the vendor.
✗No tone or formality level specified
✗No length constraint — could be 10 or 500 words
✗No context about why declining or the relationship
✗No constraints on what to avoid (apologies, etc.)
✓ Strong Prompt
Write a professional email declining a software vendor's proposal. Context: We evaluated their project management tool but chose a competitor. We want to maintain a positive relationship for future consideration. Constraints: - Tone: polite but firm - Length: under 80 words - Do NOT include apologies - Do NOT mention the competitor by name - End with an open door for future contact
✓Tone and length explicitly specified
✓Business context provided
✓Clear positive constraints (what TO do)
✓Specific negative constraints (what NOT to do)
